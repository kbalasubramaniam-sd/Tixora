Changes from v1 Feedback

1. Partner Director role removed — Product Team handles T-03 sign-off instead.
2. Partner Ops confirmed as separate Operations team (T-03 Stage 1). Additional context pending.
3. Wtheeq & Mulem access type changed from "API Only" to "Portal + API" (read-only portal by default).
4. T-02 now includes Dev Team stage for API credential creation.
5. T-03 Stage 2 changed from Partner Director to Product Team (all 3 variants).
6. T-04 now a single stage, multi-team — Dev / Integration / Business, assigned by issue context, with cross-reassignment.
7. Product Manager assignments added (from feedback FYI).
Items marked CHANGED or NEW or REMOVED are highlighted below.

T-04 · Access & Credential Support
No lifecycle change (Live partners only)

Raised by Partnership Team (Requester)

Changed: Now assignable to 3 teams based on issue type. Cross-reassignment allowed.

Stage 1
Verify & Resolve
Dev Team Integration Team Business Team
SLA: 2 business hours

Assignment Rules

Technical issue (API errors, integration bugs)→ Dev Team or Integration Team
Account issue (unlock, password reset, portal access)→ Business Team
Credential issue (API keys, certificates)→ Dev Team or Integration Team

Any assigned team can reassign to another eligible team if needed.

Issue types: Portal Login Issue (RBT, RHN) · API Credential Issue (all) · Portal Password Reset (WTQ, MLM) · Account Unlock · General Access Issue
